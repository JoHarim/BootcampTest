# 내 프로젝트 기획서 (plan.md)

> blueprint 인터뷰(2026-07-07) 결과 — 럭키런(Lucky Run) [가칭]. progress 훅이 아래 체크박스를 읽는다.
> 이전 프로젝트(사주 플래너)는 완료작: 화면은 /saju 로 이사, 설계 문서는 docs/plan/archive-saju/, 라이브는 https://joharim-saju-planner.netlify.app

## 한 문장 소개
나는 **불특정 2030 킬링타임 유저**를 위해 **스크래치 복권 + 슬롯머신 순수 운빨 도파민 게임(럭키런)** 을, 핵심 재미 **"한 판의 손맛 + 연출"** 로 만든다. 장기 목표는 스팀 유료 발매 — 이 웹 MVP 가 그 코어다.

## 데이터 (뭘 저장하나)
- localStorage 3키 (저장 어댑터 lib/game/save.ts 경유 — 스팀판 대비):
  - `luckyrun:best` 최고 기록(최다 라운드·최고 코인·누적 런)
  - `luckyrun:run` 진행 중 런(코인·라운드·목표·남은 기회·부스) — 이어하기용
  - `luckyrun:settings` 설정(소리)
- 클라우드 DB·로그인 없음. 상세: docs/plan/05-db.md

### 블럭 2. 런 시스템 + 게임장 HUD (완료 기준: 코인·라운드·목표·기회가 살아 움직이고 게임오버까지 흐른다)

- [x] 2-3. 이어하기 저장/복원 + S3 결과 화면 — 확인: 새로고침해도 런이 이어지고, 게임오버 시 결과·신기록이 뜬다 (명세: docs/plan/screens/S3-결과.md)

### 블럭 3. 스크래치 부스 (완료 기준: 티켓을 사서 긁으면 당첨금이 코인에 반영된다)

- [x] 3-2. 긁기 캔버스(45% 자동 공개) + 당첨 판정 — 확인: 긁으면 결과 공개, 당첨금 지급 (반투명 strokeStyle 로 안 긁히던 버그 수정)
- [x] 3-3. 연출(긁는 소리·당첨 효과음·컨페티) — 확인: 당첨 시 소리와 컨페티가 터진다

### 블럭 4. 슬롯머신 부스 (완료 기준: 레버를 당기면 릴이 돌고 당첨금이 반영된다)

### 블럭 5. 마감 (완료 기준: 한 런을 처음부터 끝까지 흠 없이 플레이할 수 있다)
