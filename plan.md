# 내 프로젝트 기획서 (plan.md)

> blueprint 인터뷰(2026-07-05) 결과를 옮겼다. progress 훅이 아래 체크박스를 읽어 진행상황 화면을 만든다.
> AI에게 "이 plan.md 대로 만들어줘" 하면 블럭 1의 첫 스텝부터 순서대로 빌드한다.

## 한 문장 소개
나는 **운세를 챙겨보는 2030** 를 위해 **사주 플래너(실습)** 를, 핵심 기능 **"생년월일 넣으면 오늘의 운세 보기"** 로 만든다.

## 데이터 (뭘 저장하나)
- 표: **profile (내 사주 정보)** (필드: birthDate 생년월일 필수 · birthTime 태어난 시각 선택)
- 저장 위치: 브라우저 localStorage (키 `sajuweb:profile`) — 클라우드 DB·로그인 없음
- 관계: 표가 하나뿐이라 관계 없음

## 조립 순서 (블럭 = 기능 하나 / 스텝 = 한 번에 시킬 일 하나)
> 블럭 규칙: 끝나면 눈에 보이는 결과가 하나 나온다 · 앞 블럭만 있으면 된다 · 이 블럭에 필요한 것만 만든다.

### 블럭 1. 생년월일 입력·저장 (완료 기준: 넣고 저장하면 새로고침해도 기억된다)
- [x] 1-1. 입력 화면 골격 (제목 + 빈 페이지) — 확인: 브라우저에 S1 화면이 뜬다 (명세: docs/plan/screens/S1-생년월일입력.md)
- [x] 1-2. 생년월일·시각 입력칸 + 저장 버튼 — 확인: 날짜를 고르고 저장 버튼이 눌린다
- [x] 1-3. 브라우저에 저장 + 빈 값/이상한 날짜 안내 — 확인: 저장 후 새로고침해도 값이 남아 있다, 빈 값이면 "생년월일을 입력해주세요"

### 블럭 2. 오늘의 운세 보기 (완료 기준: 저장된 생일로 오늘 일진·등급·한 줄이 뜬다)
- [x] 2-1. 저장된 생일 읽어 lunar-javascript로 오늘 일진 계산 — 확인: 화면에 오늘의 간지(예: 갑자일)가 나온다 (명세: docs/plan/screens/S2-오늘의운세.md)
- [x] 2-2. 등급 판정(오행 상생·상극) + 십신 뽑아 코멘트 + 면책 표시 — 확인: 좋음/보통/주의 중 하나와 십신 기반 한 줄, 아래 면책 문구가 보인다
- [x] 2-3. 예외 처리: 저장된 생일 없으면 S1로 보냄, 정보 수정 버튼 — 확인: 처음 진입(저장 전)엔 입력 화면으로 가고, 운세 화면의 "정보 수정"이 S1로 이동한다
