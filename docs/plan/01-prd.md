# PRD — 럭키런 (Lucky Run) [가칭]

> blueprint 인터뷰 2026-07-07 결과. 이전 프로젝트(사주 플래너)는 완료작으로 /saju 에 보존하고, 설계 문서는 docs/plan/archive-saju/ 에 있다.

## 개요 (한 문장)
나는 **불특정 2030 킬링타임 유저**를 위해 **스크래치 복권 + 슬롯머신 순수 운빨 도파민 게임**을, 핵심 재미 **"한 판의 손맛 + 연출"** 로 만든다.

## 문제 / 타겟
- 타겟: 불특정 2030 킬링타임 유저 (공개 배포 전제, 로그인 없음)
- 채우는 욕구: 짧은 시간에 긁고-돌리고-터지는 즉각적 손맛. 기록(몇 라운드까지 갔나)이 재도전 동기.
- 장기 목표: 스팀(Steam) 유료 게임. 이 웹 MVP는 그 코어 손맛의 검증기다.
  - 참고 선례: Balatro, Luck be a Landlord, CloverPit — "운빨 + 런 구조" 장르가 스팀에서 강세.
  - 스팀행 필수 확장(로드맵): 업그레이드/시너지 층, Electron/Tauri 패키징, Steam Direct 등록(USD 100).

## MVP 기능 3개
1. **스크래치 복권 부스** — 티켓 등급(동/은/금)을 골라 코인으로 구매, 캔버스를 긁어 당첨 확인.
2. **슬롯머신 부스** — 베팅 크기를 골라 레버를 당기면 릴 3개가 순차 정지. 캐비닛·레버·페이라인을 갖춘 실물형 연출.
3. **코인 지갑 + 런 구조** — 시작 코인으로 시작, 라운드마다 "제한된 플레이 기회 안에 목표 코인 달성" (스테이지 목표형). 실패·파산 = 런 종료, 기록 저장 후 새 런.

## 완성 제품 로드맵 (꿈은 층으로 보존)
- v1 (오늘): 웹 MVP — 위 3개 기능, 새 Netlify 사이트 배포
- v2: 업그레이드/아이템 시너지 층 (로그라이크화), 결과 공유
- v3: Electron/Tauri 패키징 + Steam Direct 등록 + 상점 자산

## 안 하는 것 (스코프컷)
- 로그인 / 온라인 리더보드 / 결제 / 광고 / 코인 리필 / 일일 보상 / 공유 기능
- 업그레이드·아이템 시너지 층 (v2로 보존)
- 세 번째 게임(룰렛 등)
- 관리자 화면 (확률·밸런스는 코드 상수)

## 지킬 조건 (비기능)
- PC 가로 우선 설계, 모바일 세로도 동작 (반응형)
- 사운드 기본 켬 + 음소거 버튼. 브라우저 자동재생 정책 준수 (첫 클릭 후 오디오 활성)
- 타이틀 하단 고지 한 줄: "본 게임의 코인은 가상 재화로 현금 교환·환전이 불가능합니다"
- 외부 의존 0: 시스템 폰트 + 이모지 + CSS 그래픽만 (오프라인에서도 동작)
- 저장은 localStorage, 단 저장 어댑터(load/save/clear) 한 겹으로 감싸 스팀판에서 교체 가능하게
- 저장 데이터가 깨져 있으면 조용히 새 런 취급 (에러 화면 금지)

## 밸런스 원칙 (수치는 전부 조정 가능)
- 기대수익률(EV)은 1보다 약간 낮게 (약 0.90~0.95) — 라운드 목표를 넘으려면 운과 베팅 크기 선택이 필요해진다
- 고분산 설계: 잭팟(스크래치 금 티켓 최고상, 슬롯 7 트리플)은 드물지만 화면이 뒤집어지게
- 라운드 목표 코인은 약 1.6배씩 상승, 플레이 기회는 라운드당 10회
