## 1. 사주 → 오행 점수 (PRD 5-1 승계)

- 글자 추출: 천간 4 + 지지 4 + 각 지지의 지장간. 시각 모르면 시주 제외(천간3+지지3).
- 자리별 가중치: **월지 ×3 / 일간·일지 ×2 / 그 외 천간 ×1 / 그 외 지지 ×1.5 / 지장간 각 ×0.5**
- 생극 보정: 상생 — 각 오행 점수의 **30%** 를 자기가 생하는 오행에 가산 (木→火→土→金→水→木)
  상극 — 각 오행 점수의 **20%** 만큼 자기가 극하는 오행에서 감산 (木剋土, 土剋水, 水剋火, 火剋金, 金剋木)
  (보정은 원점수 스냅샷 기준으로 동시 적용, 결과가 음수면 0으로 클램프)

## 2. 오행 점수 → 능력치 (PRD 매핑 승계)

- 木=hp(체력) / 火=atk(공격) / 土=luk(운) / 金=def(방어) / 水=int(지능)
- 변환식: `stat = round(16 + elementScore × 6)` — 시주 없으면 elementScore 합이 작아지므로
  시주 없을 때는 ×6 대신 **×7.5** (스케일 보정: 총 가중치 비율 ≈ 4:5).
- 목표 범위: 보통 사주 스탯 25~55, 극단 편중 사주 70+. (밸런스 시뮬 후 계수 조정 가능 — 상수는 파일 상단에 모아둘 것)
- 종합 전투력 `power = hp×2 + atk×3 + def×2 + int×3 + luk×1` (공개 화면 연출용)

## 3. 격국 → 직업 (PRD 5-3 단순화: 월지 본기 vs 일간의 십성)

십성 판정 규칙(플래너 lib/saju.ts 와 동일): 같은 오행=비견/겁재, 내가 생=식신/상관, 내가 극=편재/정재,
나를 극=편관/정관, 나를 생=편인/정인 (음양 같으면 앞, 다르면 뒤).

| 십성 | 직업 | 이모지 | 패시브 (statKey +%) | 스킬 방향 |
|---|---|---|---|---|
| 비견 | 무사 | ⚔️ | atk +10 | 연격(데미지) |
| 겁재 | 검투사 | 🗡️ | atk +15 | 도박 일격(고데미지) |
| 식신 | 성직자 | 🕊️ | hp +15 | 회복 |
| 상관 | 음양사 | 📿 | int +15 | 주술(데미지) |
| 편재 | 협객 | 🥷 | luk +20 | 급습(데미지) |
| 정재 | 상단주 | 💰 | luk +15 | 금전 공세(데미지) |
| 편관 | 광전사 | 🪓 | atk +20 | 광폭 일격(데미지) |
| 정관 | 수호기사 | 🛡️ | def +20 | 심판(데미지) |
| 편인 | 술사 | 🔮 | int +20 | 비전 폭발(데미지) |
| 정인 | 현자 | 📜 | int +10 | 지혜의 빛(회복) |

- 스킬 쿨다운 3턴 통일. 데미지형 위력: `int × power(≈1.6~2.2)` / 회복형: `maxHp × 0.35`.
- 이름·설명·수치 세부는 content 빌더 재량 (톤: 동양 판타지, 가볍고 경쾌).

## 4. 전투 (PRD 5-2·5-7 승계, 반자동 턴제)

- 한 스텝 = 플레이어 행동(attack|skill) → 몬스터 생존 시 반격 1회. UI가 800ms 간격 자동 스텝(방치형 느낌),
  스킬 버튼만 유저 개입.
- 플레이어: `maxHp = 40 + hp × 3` / 기본공격 데미지 = `atk × 상성배율 × 변동(0.9~1.1) − foe.def×0.35` (최소 1)
  (원안 60+hp×4·def×0.5 는 2026-07-07 시뮬에서 전원 무패로 판명 → 하향)
- 상성배율(elementMultiplier): 내 속성이 상대를 **극하면 ×1.5**, 상대가 나를 **극하면 ×0.7**, 그 외 ×1.0
- 치명타: 확률 `min(5 + luk×0.25, 35)%` → ×1.6 (토 점수가 구조적으로 높아 0.4/40은 전원 상한 도달)
- 몬스터 공격도 동일 수식(상성 포함, 치명타 없음).
- 난수: 결정적 LCG — `seed = (seed * 1103515245 + 12345) % 2^31`, `rand01 = seed / 2^31`. Math.random 금지(재현성).
- 오늘의 기운: 일운 천간 오행에 해당하는 스탯 +15% (전투 시작 시 반영, DailyFortune.desc 로 표기)

## 5. 던전·성장

- 던전 5종 (오행별): 🌲청림(木) / 🌋화염굴(火) / 🏜황토평원(土) / ⛰백철광산(金) / 🌊현무담(水)
- 각 5스테이지, 5번째 보스. 몬스터 스탯 기준(스테이지 s, 던전 난이도 d 1~5) — 2026-07-07 시뮬 튜닝 반영:
  `hp = 40 + 22×(s-1) + 14×(d-1)` `atk = 12 + 5×(s-1) + 4×(d-1)` `def = 3 + 2×(s-1)` 보스는 hp×1.5, atk×1.1 (지구전형)
- 던전 잠금 없음(어디든 입장 가능) — 상성 골라가는 재미. 스테이지는 순차 해금.
- 경험치: 몬스터 exp = `10 + 6×(s-1) + 6×(d-1)`, 보스 ×2. `expForLevel(L) = 15 + 10×(L-1)`, MAX_LEVEL 20.
- 레벨업: 모든 스탯 `round(base × (1 + 0.10×(L-1)))` (statsAtLevel).
- 시뮬 근거(78명 표본): 완주 76/78, 스테이지1 승률 100%, 커브 레벨 보스 승률 88~93%, 최약체 2명만 청림 보스에서 파밍 필요.
- 패배해도 잃는 것 없음(재도전). 클리어 기록은 던전별 최고 스테이지.

## 6. 화면 흐름 (route: /rpg — 단일 클라이언트 페이지 + 뷰 상태머신)

`loading → create(생일 입력) → reveal(능력치 공개 연출) → home(허브) → dungeon(던전 선택) → battle(전투) → result(보상) → home`

- create: 플래너 저장값(`sajuweb:profile`)이 있으면 "플래너에 저장된 생일 불러오기" 버튼 제공
- reveal: 사주 8자 카드(오행색 테두리) + 오행 점수 바 + 직업 + 전투력 — 이 화면이 임팩트 코어
- home: 캐릭터 요약 · 오늘의 기운 카드 · 던전 입장 · (하단) 면책 문구
- battle: HP바 2개 + 이모지 전투 연출 + 로그 + 스킬 버튼(쿨다운 표기) + 자동 진행
- result: 승리(경험치·레벨업·다음 스테이지) / 패배(재도전 권유)
- 저장: localStorage `sajuweb:rpg` (SaveData) — 생일 바뀌면 캐릭터 재생성

## 8. 모드 B — 사주 수집 (2026-07-08 추가, PRD 시나리오 B·5-6 승계)

- **시작**: 모드 선택(내 사주 / 사주 수집) → 수집 모드는 일간 10택(갑~계, 오행색 카드). 일간이 내 정체성(공격 속성)이자 유일한 보유 글자.
- **슬롯 7개**: 년간·년지·월간·월지·일지·시간·시지 (일간은 고정, 교체 불가). 전부 비운 채 시작.
- **점수 계산**: 모드 A와 동일 공식·가중치. 빈 슬롯은 0 기여, 지장간은 채워진 지지만. 스케일은 항상 ×6(8자 기준) — 7슬롯을 다 채우면 모드 A와 수학적으로 동일해진다.
- **격국**: 월지가 비어 있으면 **무명객(無格)** 폴백 직업. 월지를 채우는 순간 본기 십성으로 직업 각성 — 수집의 첫 마일스톤.
- **드랍**: 전투 승리 시 글자 1개 (70% 던전 오행의 천간/지지, 30% 전체 랜덤). 보스는 2개. 시드 LCG(전투 종료 시드 이어씀), Math.random 금지.
  "화 글자가 필요하면 화염굴을 돌아라" — 던전 선택 = 파밍 전략.
- **장착·교체**: 인벤토리(중복 허용, 개수 표시) → 글자 선택 → 유효 슬롯(천간은 간 자리, 지지는 지 자리)에 장착. 교체 시 기존 글자는 인벤 복귀. 장착 즉시 능력치·격국·직업·전투력 재계산 (레벨·경험치·클리어 기록은 유지).
- **완성**: 7슬롯 전부 채우면 완성 배지 (추가 도전 콘텐츠는 v3).
- **저장 v2**: `sajuweb:rpg` 에 `mode: "A"|"B"` 추가. 구버전(mode 없음) 저장은 mode "A"로 마이그레이션. 저장 슬롯은 1개 — 모드를 바꾸면 새 캐릭터(진행 리셋)임을 create 화면에 명시.
- **페이싱(2026-07-08 시뮬 실측, 10일간 전수)**: 일간 1글자 청림 s1 승률 100%, 팔자 완성까지 평균 8승(6~13), 25스테이지 완주 10/10, 완성 빌드는 모드 A 와 수학적 동치 확인. 완성 후에는 글자 교체(오행 몰아주기)가 최적화 콘텐츠가 된다.

## 9. 십성 시너지 (2026-07-08 추가, PRD 5-4 승계)

- **집계**: 일간을 제외한 최대 7글자 — 천간 3(년·월·시) 그대로, 지지 4(년·월·일·시)는 **본기**로 — 각각 일간 기준 십성을 뽑아 5그룹으로 묶는다: 비겁(비견·겁재) / 식상(식신·상관) / 재성(편재·정재) / 관성(편관·정관) / 인성(편인·정인). 빈 슬롯은 미집계 — 모드 B는 장착할 때마다 실시간 변동.
- **효과 (개수 n 선형, 상한 = 자연 상한 7)**:

| 그룹 | 이름 | 효과 |
|---|---|---|
| 비겁 ⚔️ | 형제의 기세 | 기본공격 피해 +6%×n |
| 식상 🎨 | 재주의 결 | 스킬 위력(피해·회복) +8%×n |
| 재성 💰 | 재물의 눈 | 치명타 확률 +3%p×n (시너지 포함 상한 50%) |
| 관성 🛡️ | 법도의 갑주 | 받는 피해 −4%×n (하한 ×0.6) |
| 인성 📜 | 학문의 숨 | 매 스텝 끝 최대 생명력의 1%×n 회복 |

- 수치·이름은 content.ts 의 SYNERGIES 데이터 한 곳에만 둔다(battle·UI 가 같은 소스 소비 — 드리프트 금지).
- UI: reveal·home 에 "십성 시너지" 카드 — n>0 그룹만 "식상 ×3 — 스킬 위력 +24%" 형식. 전투 화면에는 발동 중 그룹 아이콘만 컴팩트 표기.
- 밸런스 가드: 인성 회복이 몬스터 화력을 넘어서는 무한전 방지 — 시뮬에서 스텝 가드(300) 도달 케이스를 정체로 판정해 점검한다.

## 10. 신살 (2026-07-09 추가, PRD 5-5 승계)

- **판정 (표준)**: 기준지 = 채워진 일지·년지 각각. 기준지의 삼합군(申子辰/寅午戌/巳酉丑/亥卯未)에 대응하는 타깃 글자가 **기준지가 아닌 다른 기둥의 지지**에 있으면 발동. (2026-07-09 교정 — 기준지 자신 포함 판정은 화개가 72% 발동해 특별함이 사라짐. 12개월 전수 실측: 도화 29%·역마 67%·화개 26%·천을귀인 51% — 역마·천을은 흔한 대신 효과가 소소한 상시 특전, 도화·화개가 빌드 결정형.)
  - 도화(桃花): 申子辰→酉 / 寅午戌→卯 / 巳酉丑→午 / 亥卯未→子
  - 역마(驛馬): 申子辰→寅 / 寅午戌→申 / 巳酉丑→亥 / 亥卯未→巳
  - 화개(華蓋): 申子辰→辰 / 寅午戌→戌 / 巳酉丑→丑 / 亥卯未→未
  - 천을귀인(天乙貴人)은 일간 기준: 甲戊庚→丑·未 / 乙己→子·申 / 丙丁→亥·酉 / 壬癸→巳·卯 / 辛→午·寅 — 해당 지지가 사주에 있으면.
- **효과 (SINSALS 데이터 단일 소스)**:

| 신살 | 이모지 | 효과 |
|---|---|---|
| 도화 | 🌸 | 경험치 +20%, (모드 B) 글자 드랍 +1개 확률 35% |
| 역마 | 🐎 | 전투 시작 시 선제 일격 (공격력 60%, 치명 없음) |
| 화개 | 🪷 | 스킬 쿨다운 −1 (3→2) |
| 천을귀인 | 🌟 | 전투당 1회, 치명상 시 최대 생명력 30%로 부활 |

- 모드 B: 지지를 장착·교체하면 신살이 생기고 사라진다 — 신살 조합도 빌드의 일부.

## 11. 운의 흐름 — 대운·세운·월운·일운 (2026-07-09 추가, PRD 5-8 승계)
