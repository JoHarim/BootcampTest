# 유저플로우 — 럭키런

```mermaid
flowchart TD
    A[진입 /] --> B{저장된 런 있나}
    B -- 없음 --> S1[S1 타이틀: 새 런 / 기록]
    B -- 있음 --> S1b[S1 타이틀: 이어하기 + 새 런 / 기록]
    S1 -- 새 런 --> INIT[런 시작: 코인 500, 라운드 1, 목표 800, 기회 10]
    S1b -- 이어하기 --> S2
    S1b -- 새 런 --> CONFIRM{진행 중 런 버림 확인} -- 예 --> INIT
    INIT --> S2[S2 게임장: HUD + 부스]
    S2 -- 스크래치 부스 --> SC[티켓 등급 선택 → 긁기 → 당첨금]
    S2 -- 슬롯 부스 --> SL[베팅 선택 → 레버 → 릴 정지 → 당첨금]
    SC --> JUDGE{판정}
    SL --> JUDGE
    JUDGE -- 기회 남음 & 목표 미달 --> S2
    JUDGE -- 목표 달성 --> BANK{마감 버튼 활성: 마감할까 계속할까}
    BANK -- 라운드 마감 --> CLEAR[클리어 팝업: 남은 기회 x 보너스 코인, 목표 x1.6, 기회 리셋] --> S2
    BANK -- 계속 플레이 --> S2
    JUDGE -- 기회 0 & 목표 달성 --> CLEAR
    JUDGE -- 기회 0 & 목표 미달 --> S3[S3 결과]
    JUDGE -- 코인 < 50 & 목표 미달 & 대출 가능 --> RESCUE{구제 모달: 미스터 핀}
    RESCUE -- 빚내서 달린다 --> S2
    RESCUE -- 접는다 --> S3
    JUDGE -- 코인 < 50 & 목표 미달 & 대출 불가 --> S3
    S2 -- 한탕 자금 버튼 --> LOAN[대출: 코인+, 빚+, 기회마다 이자] --> S2
    S2 -- 전액 상환 버튼 --> REPAY[원금+이자 일시불] --> S2
    S3 -- 새 런 --> INIT
    S3 -- 타이틀로 --> S1
```

## 갈림길 정리
- 저장된 런이 있으면 타이틀에 이어하기가 추가로 뜬다. 이어하기는 저장 상태 그대로 S2 복귀.
- 진행 중 런이 있는데 새 런을 누르면 "지금 런을 버립니까?" 확인 후 시작.
- 목표를 미리 달성하면 "라운드 마감" 버튼이 켜진다 (허점 라운드 확정): 마감 = 남은 기회당 보너스 코인(라운드 비례)을 받고 안전하게 클리어 / 계속 = 더 벌 수도, 목표 밑으로 떨어져 죽을 수도. 이 선택이 게임의 핵심 전략이다.
- 게임오버 순간 저장된 런은 삭제되고 최고 기록만 갱신된다(빚 활성 시 기록은 순자산 기준).
- 대출 판정 이원화: 목표 달성·마감 = 순자산(코인-빚) / 파산·구매 가능 = 총 코인. 이 이원화가 "대출로 공짜 클리어"와 "대출 직후 즉사"를 동시에 막는다 (docs/plan/changes/2026-07-08-사채상어-미스터핀-대출.md).
- 이어하기 진입 시 즉시 판정 — 플레이 도중(지불 후) 저장된 런이 조작 불능으로 멈추는 것을 방지.
