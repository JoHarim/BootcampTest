# 변경 제안: 사채 상어 "미스터 핀" 대출 (한탕 시스템)

> 회장 아이디어(2026-07-08): "은행대출 — 한탕하고 싶을 수 있잖아? 대신 시간이 지나면서 이자가 쌓인다."
> 4방향 병렬 분석(밸런스 수학·악용 헌팅·선례 리서치·연출) 후 카드 문답으로 승인됨.
> 컨셉은 카드 문답에서 '은행' 대신 '사채 상어'로 확정 — 플레이당 이자는 은행이 아니라 사채 수치이고,
> 빌런이 계약의 불리함을 직접 떠벌리는 구조가 비용 고지·도박 미화 회피·카툰 톤을 동시에 만족.

## ① 뭘 바꾸나 (한 줄)
게임장에 사채 상어 "미스터 핀" 대출을 추가한다 — 빌린 돈으로 한탕을 노리되, 플레이(기회 1회)마다 이자가 쌓인다.

## ② 왜
파산 직전의 "한 번 더"와 라운드 초반의 "한탕 베팅"이라는 도박 드라마의 정점을 추가. CloverPit·Dope Wars 등 선례가 검증한 재미 축.

## ③ 규칙 (분석으로 확정된 최종안)
- 대출액 = min( round50(목표의 50%), 400 x 잔여기회 ) — R1 400, R2 650, R3 1050 … 잔여기회 캡이 "베팅도 못 하는 죽은 원금" 함정 제거
- 이자 = 기회 1회 소모마다 원금의 5% (정수 반올림, 대출 시점에 고정: R1 대출 400 → +20/기회)
- 한도 = 활성 1건 + 라운드당 신규 1회. 대출 가능 조건: 빚 없음 && 이번 라운드 미사용 && 잔여기회 ≥ 1 && 순자산 < 목표
- 판정 이원화(핵심 불변식): 목표 달성·마감 버튼 = 순자산(코인 − 빚) 기준 / 파산·구매 가능 = 총 코인 기준
  (파산까지 순자산 기준이면 대출 직후 즉사하는 자기모순 — 4개 분석 전원이 지적, 수리 완료)
- 라운드 종료(클리어·마감 공통) 정산 순서: 순자산으로 판정 → 마감 보너스 지급 → 빚 자동 전액 상환. 빚은 절대 다음 라운드로 못 넘어감(총이자 자연 상한 = 원금 100%)
- 수동 조기 상환 = 원금+누적이자 전액 일시불만(이자 증발 악용 차단). 상환 후 코인 < 50 이 되는 상환은 버튼 비활성(구제 모달 무한루프 차단)
- 구제 모달: 코인 < 50 && 목표 미달 && 대출 가능일 때 게임오버 대신 1회 제안 — 총 상환액·필요 금액 정직 표기, 거절 = 즉시 게임오버
- 신기록(bestCoins)은 순자산 기준(최소 0) — 빚 낀 기록 부풀림 방지
- 저장 하위호환: 빚 필드는 luckyrun:run 안의 옵션 필드 — 구버전 저장 런이 증발하지 않게 로드 시 정규화
