# 결정 기록 — 사주 플래너 (실습)
> 방향을 정하거나 바꾼 결정을 위에서부터 쌓는다(최신이 위). 한 결정 = 4줄.
> 왜 남기나: 2주 뒤의 나(와 새 AI 세션)가 "왜 이렇게 만들었지?"를 다시 고민하지 않게.

## [2026-07-07] 배포는 정적 내보내기(output: export) + Netlify CLI
- 왜: 앱이 서버 기능 없이 브라우저에서만 돌아서(localStorage) 정적 파일이면 충분. Netlify CLI가 이미 로그인돼 있어 무인 배포 가능.
- 버린 대안: Vercel 웹 콘솔 Import(수동 단계 필요) / Netlify Next.js 런타임(서버 기능 없는데 무거움).
- 영향: `apps/web/next.config.mjs`에 `output: "export"` 추가 → 빌드 시 `apps/web/out/` 생성 → `netlify deploy --prod --dir out`. 이후 서버 기능(API·SSR)을 쓰게 되면 이 설정을 빼고 배포 방식을 바꿔야 함.

## [2026-07-05] 저장은 브라우저(localStorage), Supabase를 쓰지 않는다
- 왜: 첫 실습이라 로그인·키·계정 없이 바로 돌아가는 게 우선. 단일 사용자·개인용이라 클라우드가 필요 없음.
- 버린 대안: Supabase 클라우드 저장 — 계정·키 준비가 한 단계 더 들어 첫 실습엔 부담.
- 영향: `.env` 키 불필요. 기기·브라우저 바뀌면 데이터 사라짐. 다음 버전에서 supabase 어댑터로 전환 예정.

## [2026-07-05] 운세 계산은 진짜 사주(lunar-javascript), 해석은 오행 상생·상극 + 십신
- 왜: 진짜 사주 계산을 선택. 만세력은 직접 짜지 않고 무료 라이브러리에 맡김. 등급은 오행 관계로, 그날의 성격은 십신(십성)으로 더 사주답게.
- 버린 대안: (1) 규칙 기반 가벼운 운세 (2) AI가 운세 생성 (3) 오행 상생·상극만 — 각각 진짜 사주가 아니거나, 십신 없이 뭉뚱그려짐.
- 영향: (a) 등급 = 내 일간 오행 vs 오늘 일진 오행 (상생=좋음 / 같음=보통 / 상극=주의) (b) 그 위에 오늘 일진 천간의 십신(일간 기준, 음양까지)을 뽑아 코멘트·등급 보정. 십신→등급/문구 표는 기본값으로 시작해 조정. 계산이 늘어 블럭 2가 조금 커짐.

## [2026-07-05] MVP는 "오늘의 운세 보기"까지, 계획 기능은 다음 버전
- 왜: 첫 실습은 작게. 생년월일 입력 → 오늘 운세 한 화면이 돌아가는 걸 먼저 본다.
- 버린 대안: 오늘의 할 일 계획(To-Do)을 MVP에 포함 — 화면·저장이 늘어 첫 실습엔 과함.
- 영향: 창고는 profile 표 하나. 계획·지난 운세·궁합은 로드맵으로 보존.

## [2026-07-05] 로그인·결제·AI·공유는 넣지 않는다
- 왜: 단일 사용자 개인 실습. 인증·과금 없이 바로 쓰는 게 목표.
- 버린 대안: 소셜 로그인·AI 상담 — 키·비용·복잡도 상승.
- 영향: 문(외부 서비스) 0개. 보안 부담 최소.
