# PRD — 사주 플래너 (실습)
> 작성일: 2026-07-05 · blueprint 인터뷰 기반 · 상태: 초안

## 1. 개요
- 한 줄 정의: 내 사주로 오늘의 운세를 보고, 그에 맞춰 하루를 계획하는 플래너 (MVP는 "오늘의 운세 보기"까지)
- 레퍼런스: 시중 사주·운세 앱 (다른 점: 광고·복잡함 없이, 생년월일 한 번만 넣으면 매일 오늘 운세를 바로 본다)
- 배경·동기: 운세를 봐도 "그래서 오늘 뭘 하지"가 없다 → 보기에서 계획으로 잇는 게 목표 (계획 기능 자체는 다음 버전)
- 성공 기준: 생년월일을 넣으면 오늘의 일진(그날의 간지)과 운세 등급이 화면에 뜬다. 새로고침해도 내 정보가 기억된다.

## 2. 문제·타겟
- 페르소나: 운세를 챙겨보는 20~30대
- 문제 장면: 매일 운세를 찾아보지만 앱마다 광고가 많고, 봐도 행동으로 이어지지 않는다
- 현재 대안과 한계: 포털 운세·사주 앱 — 광고 많음, 결과만 툭 던짐, 계획과 단절
- 사용자 = 결제자 (본인, 무료) · 규모 가늠: 개인·실습용

## 3. 기능 (MVP)
| 순위 | 기능 | 완료 기준 |
|---|---|---|
| 1 | 생년월일 입력·저장 | 생년월일(선택: 태어난 시각)을 넣고 저장하면 브라우저에 기억된다 |
| 2 | 오늘의 운세 보기 | 저장된 생일로 오늘의 일진 간지 + 운세 등급 + 한 줄 코멘트가 뜬다 |

### 완성 제품 로드맵 (꿈은 보존, 층으로 쌓는다)
- 다음 버전 (감당 가능, 이번엔 안 함): 오늘의 할 일 계획(To-Do), 지난 운세 모아보기, 띠·간단 궁합
- 완성 제품 (최종 그림): 월운 달력 · 대운 흐름 · AI 사주 상담까지 붙인 "운세 + 계획" 통합 플래너
- 감당선 밖: 없음 (MVP는 무료 라이브러리로 감당 가능)

### Out of Scope (절대 안 함)
- 로그인, 결제, AI 상담, 타인과 공유 — 이번 MVP 밖

## 7. 외부 연동
- 기본 스택(Supabase·Vercel)은 이번 MVP에서 쓰지 않는다 — 저장은 브라우저(localStorage), 배포는 이번 스코프 밖.
- 외부에서 빌리는 "문(門, 외부 서비스)"은 없다.
- 재료(주방에 놓고 쓰는 무료 라이브러리): `lunar-javascript` (npm, MIT 라이선스, API 키 불필요) — 생년월일·날짜를 간지로 계산.

| 기능 | 플랫폼 | 발급처 | .env 키 |
|---|---|---|---|
| 사주 간지 계산 | lunar-javascript (라이브러리) | npm | 없음 |
| 저장 | 브라우저 localStorage | — | 없음 |

## 8. 지킬 조건 (개발자 말로 '비기능' — 앱이 하는 일이 아니라 지켜야 할 규칙)
- 권한 등급: 단일 사용자, 로그인 없음 (내 데이터만 이 브라우저에)
- 관리자: 없음
- 최우선 보호 데이터: 생년월일 — 브라우저에만 저장, 외부로 전송하지 않음
- 실패 시 안내: 입력 검증(빈 값·이상한 날짜)·계산 실패 시 앱이 죽지 않고 안내
- 면책: "운세는 재미로 참고해주세요" 한 줄을 운세 화면에 표시
- 문의 채널: 없음 (실습) · 백업: 없음 (브라우저 저장이라 기기·브라우저 바뀌면 사라짐 — 다음 버전에서 Supabase 전환 시 해결)
